├── fixtures/                       # Test fixtures
│   └── auth.json                  # Saved authentication state (auto-generated)
│
├── .env                           # Environment variables (BASE_URL, credentials)
├── playwright.config.js           # Playwright configuration
├── package.json                   # Dependencies
└── README.md                      # Complete documentation

/**
 * ====================================================================
 * 2. AUTHENTICATION FLOW
 * ====================================================================
 */

Step 1: login.setup.js runs FIRST (setup project dependency)
        ↓
        Logs in using process.env.USER_EMAIL and process.env.USER_PASSWORD
        ↓
        Saves session to fixtures/auth.json
        ↓
Step 2: All test specs (login.spec.js, users.spec.js) load auth.json
        ↓
        Tests run with saved session (NO re-login needed)

/**
 * ====================================================================
 * 3. TEST EXECUTION ORDER
 * ====================================================================
 */

1. setup project (login.setup.js)
   - Creates fixtures/auth.json
   
2. chromium project (depends on setup)
   - login.spec.js (3 test cases)
   - users.spec.js (9 test cases)

// Run in headed mode
npx playwright test --headed

// Generate report
npx playwright show-report

/**
 * ====================================================================
 * 10. MAINTENANCE & EXTENSION
 * ====================================================================
 */

Adding New Tests:
1. Add selectors to selectors/*.selectors.js
2. Add methods to pages/*.page.js
3. Create test cases in tests/*.spec.js
4. Add test data to utils/userData.json (if needed)

Adding New Pages:
1. Create selectors/newpage.selectors.js
2. Create pages/newpage.page.js
3. Create tests/newpage.spec.js
4. Update playwright.config.js if needed

/**
 * ====================================================================
 * 11. BEST PRACTICES IMPLEMENTED
 * ====================================================================
 */

1. Separation of Concerns
   - Tests focus on scenarios
   - Pages handle interactions
   - Selectors are isolated
   
2. DRY Principle (Don't Repeat Yourself)
   - Reusable methods in page objects
   - Common selectors in selector files
   
3. Data-Driven Testing
   - Test data in external JSON files
   - Environment-based configuration
   
4. Authentication Optimization
   - Login once, reuse session
   - Faster test execution
   
5. Clean Code
   - Meaningful variable/method names
   - Comprehensive comments
   - Consistent formatting
